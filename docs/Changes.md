# Deviations from Design I
Several features from the 'Design I' phase were either modified or omitted in Cycle I. This document details the motivations and reasoning behind those changes to the original plan.

_This document concerns smaller changes made to the design plan discussed during meetings or over messages between team members (before narrowing of the scope of the entire project). For documentation on narrowing of the overall scope of the project, see [ScopeChanges.md](ScopeChanges.md)._

**_Disclaimer:_**
_For all of Design I and most of Implementation I, this document had not been updated. This is because the information from the meetings/messages was recorded on personal computers. Moving forward, we will update this document actively._

## Omitted
#### User Story R-2: Registrars can view analytics of classrooms [See GitLab Issue #22]
**Reasoning:** Analytics were deemed to be unecessary due to the small scale of the application, as discussed during [Meeting 4](CHANGELOG.md). \
**Current Status:** Removed from design of system entirely. 

#### User Story R-4: Registrars can view ratio of students compared to maximum capacity of classroom [See GitLab Issue #24]
**Reasoning:** Implementation would require staff to enter class sizes when booking, which was decided to be outside of the requirements baselined during Design I over messages between team members. \
**Current Status:** Removed from design of system entirely.
